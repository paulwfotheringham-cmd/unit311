Client intake sequence

sequenceDiagram
    participant C as Client
    participant P as /whatsapp/support or support-flow
    participant API as /api/whatsapp/inbound
    participant I as support-intake
    participant DB as Supabase
    participant M as Support channel

    C->>P: Open new ticket
    P->>API: preview POST
    API->>I: step prompts
    I-->>C: What is your name?
    C-->>I: name
    I-->>C: organisation / priority / description
    C-->>I: answers
    I->>DB: Create SUP-xxx
    I->>M: New ticket + assignment prompt
    I-->>C: Thank you. Agent will be assigned shortly.

Assignment (platform only — not client)

sequenceDiagram
    participant Op as Internal operator
    participant Msg as Messaging → Support
    participant API as assign API / claims handler
    participant DB as support_tickets
    participant CMB as CallMeBot
    participant Client as Client WhatsApp

    Op->>Msg: Join as User 1
    Op->>Msg: User 1
    Msg->>API: Claim message
    API->>DB: user_assigned = User 1
    API->>Msg: System: User1 assigned to SUP-xxx
    API->>CMB: Operator alert on phone
    API->>Client: Support ticket SUP-xxx created. User1 assigned…

Client never assigns themselves. Assignment happens only from Messaging → Support or the support-flow demo panel 3.

Updates & close ticket

sequenceDiagram
    participant Op as Support workspace / flow panel 4
    participant API as client-update / close API
    participant CMB as CallMeBot
    participant C as Client WhatsApp

    Op->>API: Send update text
    API->>CMB: Exact update message
    CMB->>C: Status update

    Op->>API: Close ticket
    API->>DB: closed = true
    API->>CMB: Ticket SUP-xxx is now closed.
    CMB->>C: Close notification
